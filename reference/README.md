# reference/ — Эталонные данные для тестирования

## Что сюда класть

Положи сюда PDF файл заключения эксперта:

```
reference/
├── zakluchenie.pdf          ← ← ← ВОТ СЮДА (любое имя .pdf)
├── expert_conclusion.json   (генерируется автоматически)
└── expert_conclusion.txt    (генерируется автоматически, для чтения)
```

## Как подготовить эталон

```bash
# 1. Подготовить эталон из PDF:
python tools/parse_conclusion.py reference/zakluchenie.pdf

# 2. Проверить что извлеклось правильно:
cat reference/expert_conclusion.txt
```

## 4 тестовых сценария

| Сценарий | Что подаёшь боту | Ожидание |
|---|---|---|
| **С1 — Сырая подача** | ZIP "Документация предоставленная" | Бот найдёт ошибки (расхождения с экспертом — нормально) |
| **С2 — Сравнение** | Любой архив | Прямое сравнение вердикта бота с заключением |
| **С3 — Исправленная** | ZIP "Документация откорректированная" | Ошибок меньше, чем в С1 |
| **С4 — Идеал** | ZIP "Документация окончательная" | Полное совпадение с экспертом |

## Автоматическое сравнение

```bash
# Запуск бота с автосравнением:
./start.sh --test

# Ручное сравнение отчёта бота с экспертом:
python tools/compare_with_expert.py reports/bot_report.json --scenario С2

# Watch-режим (сравнивает каждый новый отчёт автоматически):
python tools/compare_with_expert.py --watch
```
